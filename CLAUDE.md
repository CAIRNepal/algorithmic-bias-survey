## Project Overview

Research companion repo for the paper "Towards FAIR AI: A Survey of Regional Trends and Knowledge Graph-Enhanced Bias Mitigation." Contains analysis scripts for the survey corpus, a metadata extraction notebook, and an interactive Next.js dashboard deployed to GitHub Pages.

## Repository Structure

- `evaluation/analysis/` â€” Python scripts that generate figures from the survey corpus CSV
- `dashboard/` â€” Next.js 15 (TypeScript, Tailwind, Recharts, Papa Parse, react-simple-maps) interactive dashboard
- `generate_atlas_umap.py` â€” Generates UMAP embeddings + HDBSCAN clusters â†’ `dashboard/public/atlas_data.csv`
- `extract_publication_metadata.ipynb` â€” Jupyter notebook that fetches metadata from OpenAlex API given DOIs
- `figures/` â€” Output figures from original 99-paper corpus
- `evaluation/analysis/figures_new/` â€” Output figures from expanded corpus

## Data Files

Two corpus versions exist:
- `evaluation/analysis/papers.csv` â€” Original 99-paper corpus
- `evaluation/analysis/papers_new.csv` â€” Expanded corpus (519+ papers, actively maintained)

**papers_new.csv columns:** `SN, Paper Title, DOI, Authors, Author Regions, Affiliations, Year, Focus Region, Domain, Source`

**atlas_data.csv columns** (generated by `generate_atlas_umap.py`):
`SN, title, DOI, domain, year, authors, author_regions, abstract, keywords, cited_by_count, is_oa, oa_status, oa_url, openalex_countries, openalex_authors, umap_x, umap_y, umap_x3, umap_y3, umap_z3, cluster, cluster_label`

**CSV quoting:** Titles with commas must be properly quoted. When writing CSVs, use `quoting=csv.QUOTE_ALL` in pandas to avoid breaking the dashboard's Papa Parse parser.

**Affiliations column** is empty in `papers_new.csv` â€” the `top_n_university_domain_*.py` scripts cannot run until it's populated.

**Focus Region** is empty for new papers (Source=New). The dashboard no longer requires it for filtering but some analysis scripts use it.

## Adding New Papers (full pipeline)

When new papers are added to `papers_new.csv`, ALL of the following must be updated:

```bash
# 1. Add rows to source CSV
# Edit evaluation/analysis/papers_new.csv

# 2. Re-run UMAP pipeline (generates new atlas_data.csv with embeddings for all papers)
python generate_atlas_umap.py

# 3. Update dashboard/public/papers.csv (for Research Analytics page)
python -c "
import pandas as pd, csv
df = pd.read_csv('evaluation/analysis/papers_new.csv')
df.to_csv('dashboard/public/papers.csv', index=False, quoting=csv.QUOTE_ALL)
"

# 4. Commit and push â†’ GitHub Actions auto-deploys to GitHub Pages
git add dashboard/public/ evaluation/analysis/papers_new.csv
git commit -m "Add N new papers, re-run UMAP"
git push
```

**Why regenerating UMAP is required:** The Atlas uses sentence-transformer embeddings (`all-MiniLM-L6-v2`) + UMAP for 2D/3D layout and HDBSCAN for semantic clusters. Adding papers changes the embedding space and cluster assignments, so the whole pipeline must re-run.

### Atlas data flow (AbstractAtlas.tsx internals)

```
atlas_data.csv (Papa Parse)
  â†’ papers[]  (AtlasPaper type)
  â†’ filteredPapers  (query + domain + cluster + lasso + country + author + year + OA filters)
  â†’ Canvas 2D render (toScreen2 â†’ pixel coords with pan/zoom transform)
  â†’ Canvas 3D render (project3 â†’ perspective projection with arcball rotation)
  â†’ worldMapData  (country â†’ {count, domains, dominant})
  â†’ collabData    (country pair â†’ count)
  â†’ yearDist      (year â†’ count)
  â†’ countryCounts, authorCounts  (right-panel filter lists)
```

**Stale-closure fix:** `render2` and `render3` are captured in `render2Ref`/`render3Ref` refs updated every render. All RAF loops and the 3D auto-rotate loop call `render2Ref.current()` / `render3Ref.current()` â€” never the stale initial closure.

### Deployment

Automatic via `.github/workflows/deploy.yml` on push/merge to `main`. Builds the dashboard as a static export and deploys to GitHub Pages.

Production builds use `basePath: /algorithmic-bias-survey` for GitHub Pages. The atlas base path is set in the CSV fetch:
```ts
const base = process.env.NODE_ENV === 'production' ? '/algorithmic-bias-survey' : '';
Papa.parse(`${base}/atlas_data.csv`, ...);
```
